ЖУРНАЛ НАЛЬОТУ · А1-СМ «Фурія»
================================

ЯК ВИКОРИСТОВУВАТИ
1. Розпакуйте архів.
2. Відкрийте файл index.html у браузері (найкраще Chrome або Edge)
   — двічі клацніть по ньому або перетягніть у вікно браузера.
3. Введіть PIN на екрані входу (типовий PIN: 4263).
4. Вводьте дату, хвилини нальоту й примітку → «Зберегти наліт».

ВХІД (PIN)
- Перед роботою застосунок просить PIN-код.
- Після успішного входу браузер запам'ятовує, що ви залогінені
  (ознака входу зберігається в localStorage), тож наступного разу
  PIN вводити не потрібно — аж до натискання кнопки «Вийти».
- ЯК ЗМІНИТИ PIN: відкрийте index.html у текстовому редакторі,
  знайдіть рядок  const ACCESS_PIN = '4263';  і замініть 4263 на свій код.
- УВАГА: це простий захист від випадкового відкриття, а НЕ справжня
  безпека — код PIN видно у вихідному коді сторінки. Не зберігайте тут
  таємну інформацію.

ДЕ ЗБЕРІГАЮТЬСЯ ДАНІ
- За замовчуванням дані зберігаються в пам'яті браузера (localStorage)
  у форматі JSON.
- Вони лишаються після закриття сторінки на ЦЬОМУ пристрої та в ЦЬОМУ браузері.
- Додатково дані можуть АВТОМАТИЧНО зберігатися в зовнішній файл data.json
  на диску (див. нижче) — тоді кожна зміна пишеться і в localStorage, і у файл.

ЗОВНІШНІЙ ФАЙЛ ЗА ЗАМОВЧУВАННЯМ: data.json (Chrome / Edge)
- У теці застосунку є файл data.json — типове місце для збереження даних.
- ЯК УВІМКНУТИ (один раз):
  у блоці «Синхронізація з файлом» натисніть «🔗 Підключити data.json»
  і в діалозі виберіть файл data.json із теки застосунку.
- Після цього КОЖНА зміна (додавання, видалення, імпорт, очищення)
  автоматично записується в data.json.
- Браузер пам'ятає файл між сесіями. Після перезапуску браузера він може
  попросити підтвердити доступ — натисніть «🔓 Відновити доступ» (один клік),
  і автозбереження продовжиться.
- «🆕 Інший файл…» — обрати або створити інший .json замість data.json.
- «⤓ Відключити»   — припинити запис у файл (дані лишаються в localStorage).
- ПОРАДА: покладіть data.json у теку iCloud / Google Drive / Dropbox —
  так дані синхронізуватимуться між пристроями.
- ЧОМУ ПОТРІБЕН КЛІК: браузер із міркувань безпеки не дозволяє сторінці
  писати у файл без вашого підтвердження. Тому файл треба підключити
  (повністю без кліків — лише з власним локальним сервером).
- Якщо браузер не підтримує цю функцію (наприклад, Safari чи Firefox),
  блок підкаже про це; користуйтесь Експорт / Імпорт нижче.
- Якщо підключення не спрацьовує при подвійному кліку (file://),
  запустіть локальний сервер у теці застосунку, напр.:
      python3 -m http.server 8000
  і відкрийте  http://localhost:8000/index.html

ПЕРЕНЕСЕННЯ МІЖ ПРИСТРОЯМИ (Експорт / Імпорт)
- Кнопка «Експорт JSON» зберігає файл naliot_РРРР-ММ-ДД.json з усіма записами.
- Кнопка «Імпорт JSON» завантажує цей файл на іншому пристрої.
  Під час імпорту можна або ДОДАТИ записи до наявних, або ЗАМІНИТИ всі.
- Рекомендація: періодично робіть експорт як резервну копію.

СТАТИСТИКА
- Усього нальоту, кількість польотів, наліт за поточний тиждень (з понеділка),
  середня тривалість польоту та діаграма за останні 6 місяців.

УВАГА
- Якщо браузер очистити (видалити дані сайтів / приватний режим), записи
  в localStorage зникнуть, і доведеться вводити PIN знову.
  Тримайте свіжий експорт JSON або підключений зовнішній файл про запас.
