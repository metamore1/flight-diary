# Журнал нальоту · Фурія

React-застосунок журналу нальоту БпАК А1-СМ. Дані зберігаються у JSON-файлі
`data/flights.json` на сервері — вони не губляться під час перезапуску чи деплою.

## Запуск локально (розробка)

```bash
npm install
npm run dev
```

Відкрити http://localhost:5173 (API-сервер піднімається паралельно на порту 3210).

## Деплой на сервер (Node.js)

```bash
npm install
npm run build     # збірка фронтенду в dist/
npm start         # Express віддає dist/ і API, порт з $PORT або 3210
```

На сервері мають лишатися: `dist/`, `server.js`, `src/seed/flights.json`,
`node_modules/` (або `npm install --omit=dev`), `package.json` та тека `data/`.

## Де живуть дані

- **`data/flights.json`** — робочий файл журналу. Створюється автоматично з
  `src/seed/flights.json` при першому запуску сервера. Усі зміни (додавання /
  видалення польотів, імпорт) пишуться сюди атомарно.
- Під час оновлення застосунку на сервері **не перезаписуйте теку `data/`** —
  саме в ній накопичений журнал.

## Формат JSON

```json
{
  "name": "Фурія",
  "aircraft": "БпАК А1-СМ",
  "baselineMinutes": 2238,
  "baselineNote": "попередній сумарний наліт",
  "flights": [
    { "id": "f001", "date": "2026-05-19", "minutes": 144, "type": "day", "note": "" }
  ]
}
```

`type` — `day` або `night`. Рекорд, середній політ, тижневі підсумки й
зірочка за 200 год обчислюються автоматично.

## API

- `GET /api/flights` — повертає весь журнал
- `PUT /api/flights` — замінює журнал (з валідацією; некоректні дані → 400)

## Якщо залити на статичний хостинг (без Node)

Застосунок автоматично переходить у **локальний режим**: дані читаються з
вбудованого seed і зберігаються в localStorage браузера, а в шапці зʼявляється
попередження. Для резервної копії та перенесення даних є кнопки
**Експорт JSON / Імпорт JSON**. Щоб дані жили на сервері — потрібен Node-хостинг
(`npm start`).
